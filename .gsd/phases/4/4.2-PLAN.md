# Plan 4.2: Frontend Chat Component & Integration

## Objective
Build the chat UI and integrate real-time subscription using Supabase Broadcast so users can send and receive messages instantly.

## Context
- .gsd/SPEC.md
- apps/web/package.json
- apps/web/src/components/chat/ChatBox.tsx (to create)
- apps/web/src/components/dashboard/MessagesView.tsx

## Tasks

<task type="auto">
  <name>Create ChatBox Component</name>
  <files>apps/web/src/components/chat/ChatBox.tsx</files>
  <action>
    - Ensure `@supabase/supabase-js` is installed in `apps/web`.
    - Create a real-time `ChatBox` component.
    - On mount, it should fetch historical messages via `GET /api/chat/:chatId/messages`.
    - It should initialize a Supabase client using `process.env.NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
    - It should subscribe to the broadcast channel `chat:${chatId}`. On receiving a payload, append it to the local messages state.
    - Implement a message input area that calls `POST /api/chat/:chatId/messages` using `apiFetch`.
    - Render text-only messages, styling sender vs receiver bubbles differently.
  </action>
  <verify>grep "subscribe" apps/web/src/components/chat/ChatBox.tsx</verify>
  <done>ChatBox successfully fetches history, subscribes to Broadcast, and allows sending messages.</done>
</task>
